## Project Overview

**TiG Stack** is an automated deployment and management tool for a monitoring stack based on **Telegraf + InfluxDB + Grafana**, running via Docker Compose. It targets sysadmins monitoring servers, network devices (switches, routers, firewalls), UPS units, and VMware ESXi hosts via SNMP.

Two main entry points:
- **`tig-setup.sh`** — one-shot installer: detects OS, installs Docker, generates all config files, starts services
- **`tigadd.sh`** — ongoing device management CLI (`add`, `list` subcommands)

## Commands

```bash
# Initial deployment (requires sudo for Docker installation and firewall)
chmod +x tig-setup.sh && sudo ./tig-setup.sh

# Add a monitored device
./tigadd.sh add --type switch --name core-sw-01 --ip 192.168.1.1 \
  --snmp-version v2c --community public

# Add with SNMPv3
./tigadd.sh add --type server-linux --name web-srv-01 --ip 10.0.1.5 \
  --snmp-version v3 --sec-name monitor \
  --auth-pass "AuthPass123!" --priv-pass "PrivPass123!"

# Preview config without writing to disk
./tigadd.sh add --type ups --name ups-dc01 --ip 10.0.0.200 \
  --snmp-version v1 --community public --dry-run

# List devices and live status
./tigadd.sh list

# Docker operations
docker compose ps
docker compose logs telegraf --tail=50 -f
docker compose restart telegraf
docker compose down           # stop
docker compose down -v --remove-orphans  # full reset (destroys data)
```

# Remove a monitored device
./tigadd.sh remove --name core-sw-01          # prompts for confirmation
./tigadd.sh remove --name core-sw-01 --force  # skip confirmation
```

## Testing

Tests use [bats-core](https://github.com/bats-core/bats-core). Install it first:

```bash
# macOS
brew install bats-core

# Linux (via npm)
npm install -g bats

# Or clone manually
git clone https://github.com/bats-core/bats-core.git ~/.bats
export PATH="$HOME/.bats/bin:$PATH"
```

Run tests:

```bash
# All tests
bats tests/

# Single file
bats tests/test_validate.bats

# Verbose (shows test names)
bats --verbose-run tests/
```

Test files and what they cover:

| File | Covers |
|------|--------|
| `tests/test_validate.bats` | All validation rules in `lib/validate.sh` |
| `tests/test_gen.bats` | Config output structure, OID selection per device type, file numbering |
| `tests/test_add.bats` | File creation, `--dry-run`, `--force` overwrite, v3 content |
| `tests/test_remove.bats` | File deletion, confirmation prompt, error cases |

Tests stub out `_telegraf_reload` via `NO_RELOAD=true` — docker is never called during tests.

## Architecture

All bash scripts use `set -euo pipefail`. `tigadd.sh` sources library modules from `lib/` and dispatches to subcommands.

### `lib/` module layout

| File | Role |
|------|------|
| `lib/common.sh` | Color helpers, `log`/`warn`/`die` functions, InfluxDB config auto-detection |
| `lib/validate.sh` | Input validation for device type, IP, SNMP version, SNMPv3 auth/priv params |
| `lib/agent.sh` | Builds the `[[inputs.snmp]]` TOML agent block with version-specific credential fields |
| `lib/gen.sh` | Assembles full device config (agent block + OIDs), auto-numbers output filenames |
| `lib/cmd/add.sh` | `tigadd add` — validates args, calls gen.sh, writes config to `telegraf-config/telegraf.d/` |
| `lib/cmd/list.sh` | `tigadd list` — parses existing device configs, queries InfluxDB via Flux, prints table |
| `lib/oids/system.sh` | SNMPv2-MIB system OIDs (sysName, sysDescr, sysUpTime) — included by all device types |
| `lib/oids/if_mib.sh` | IF-MIB 64-bit interface counters — for switch, router, firewall, server types |
| `lib/oids/host.sh` | HOST-RESOURCES-MIB + UCD-SNMP-MIB CPU/memory/disk — for server-linux, environment |
| `lib/oids/ups.sh` | UPS-MIB (RFC 1628) battery and power OIDs |
| `lib/oids/esxi.sh` | VMware ESXi SNMP OIDs — VM count, host CPU/mem, virtual NICs |

### Config file generation flow

1. User runs `tigadd.sh add` → `lib/cmd/add.sh` validates → `lib/gen.sh` assembles TOML
2. `lib/gen.sh` sources `lib/agent.sh` (agent block) + relevant `lib/oids/*.sh` files
3. Config written to `telegraf-config/telegraf.d/10x-snmp-<name>.conf` (auto-incremented)
4. Telegraf picks up new files on restart (`docker compose restart telegraf`)

### Generated file locations (not committed)

- `.env.influxdb-admin-username/password/token` — credentials used by Docker secrets
- `docker-compose.yml` — generated by `tig-setup.sh`
- `telegraf-config/telegraf.conf` — main agent config
- `telegraf-config/telegraf.d/000-influxdb.conf` — InfluxDB output plugin
- `telegraf-config/telegraf.d/100-inputs.conf` — host metric inputs
- `telegraf-config/telegraf.d/10x-snmp-*.conf` — per-device SNMP inputs

### Supported device types

`switch`, `router`, `firewall`, `server-linux`, `server-windows`, `ups`, `esxi`, `environment`

Each type maps to a specific combination of OID modules in `lib/cmd/add.sh`.

## Key conventions

- Config files in `telegraf-config/telegraf.d/` follow `NNN-<purpose>.conf` numbering — SNMP device files start at `101` and auto-increment
- InfluxDB is queried via Flux (not InfluxQL) — the token and endpoint are read from `.env.influxdb-*` files by `lib/common.sh`
- SNMPv3 defaults: auth protocol SHA-256, priv protocol AES-128 (defined in `lib/common.sh`)
- Some inline comments in `lib/oids/esxi.sh` are written in Thai
